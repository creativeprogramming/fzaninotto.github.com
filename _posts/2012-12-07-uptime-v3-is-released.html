The master branch of the uptime repository now points to version 3, and I just tagged v3.0 on the GitHub repository. This marks the official release of Uptime V3. If you want to know more about this new release, check the previous post on the subject.

If you have an existing MongoDB database with Uptime 2.0 data, you have to migrate it before starting the application again. It takes a single command:

> node models/migrations/upgrade2to3

This is a major milestone for uptime, in usability, performance, and features. I hope you'll love it.
